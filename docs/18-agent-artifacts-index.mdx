# Agent artifacts index

Files for AI/coding agents. Keep accurate and greppable.

## Agent quick start

- **Next:** Read [Coding agent strategy](19-coding-agent-strategy).
- **Loop:** GSD (Codex) via `$gsd-*` skills for phases; `pnpm gsd:install` to install. Use `forge-loop` for `doctor`, `progress`, `sync-legacy`. `FORGE_LOOP_ID` unset → default (`.planning/`).
- **Runtime:** Codex + GSD skills for planning/phase execution; prompt-pack optional.
- **Headless gate:** validate env readiness with `forge-env doctor --mode headless --strict`.
- **GUI command center:** `forge-repo-studio open --view env` for env triage, allowlisted commands, and loop status.

**Using Cursor with this repo:** Run `pnpm gsd:install` from repo root so Codex (when used) has GSD skills in this project. In Cursor, follow AGENTS.md and `.planning`; use `forge-loop doctor`, `forge-loop progress`, and `forge-loop sync-legacy` as needed. The `.cursor/rules/planning-and-loop.mdc` rule aligns Cursor with the same artifact contract (STATE, DECISIONS, TASK-REGISTRY) as GSD/Codex.

- **`.planning/PROJECT.md`** - Product intent and boundaries.
- **`.planning/REQUIREMENTS.md`** - Requirement baseline and traceability.
- **`.planning/ROADMAP.md`** - Phase list, dependencies, plan progression.
- **`.planning/STATE.md`** - Current lifecycle position and active work.
- **`.planning/DECISIONS.md`** - Decision log (primary).
- **`.planning/ERRORS.md`** - Non-repeatable failures and fixes (primary).
- **`.planning/TASK-REGISTRY.md`** - Backlog registry (primary).
- **`.planning/TEMP-REFACTOR-BACKLOG.md`** - Temporary migration/refactor queue.
- **`.planning/HUMAN-TASKS.md`** - Human-only tasks (repos, secrets, Vercel, npm, OAuth); check before assuming a task is blocked on human action.
- **`.planning/PLATFORM-PRD.md`** - Platform product and gateway roadmap (Repo Studio backend, proxy APIs, capabilities).
- **`.planning/PRD.md`** - Default loop product intent (when present).
- **`.planning/MONETIZATION-STRATEGY.md`** - Iteration doc: revenue model, license, subscription gating, extension gating, platform entitlements; not MIT.
- **`.planning/ASSISTANT-RUNTIME-STRATEGY.md`** - Design: Forge/Codex divergence, tool unification, data stream protocol, Phase 19-04.
- **`.planning/DEFINITION-OF-DONE.md`** - Stage DoD (Planning, Execution, Review, PRD scope); use to determine when a stage is complete.
- **`.planning/phases/*`** - Context, research, plans, summaries, UAT, verification. Phase 20 wires DoD and HUMAN-TASKS into Repo Studio Planning workspace (snapshot, panels, unified todos); see ROADMAP.
- **`.planning/prompts/*`** - Runtime-agnostic prompt packs for coding-agent/manual runs.
- **`.planning/migration-report.json`** - One-time migration report from legacy artifacts.

## Expected repo layout

Monorepo: **root** contains `.planning/`, `apps/`, `packages/`, `vendor/`, `docs/`, `scripts/`. **No top-level `app`** — applications live under `apps/*` (e.g. repo-studio, docs, platform). **Repo Studio** is the primary app for Forge Loop. **Vendor:** submodules and vendored deps (e.g. `vendor/get-shit-done`, `vendor/repo-studio-extensions`, `vendor/twick`; Phase 16/17 add vendor/repo-studio, vendor/platform). **Extensions:** installables and examples in `vendor/repo-studio-extensions`; per-project config at `.repo-studio/extensions`. **Docs:** how-tos and agent-artifacts in `docs/`; docs site app in `apps/docs`. **Legacy:** `docs/agent-artifacts/core/*` are read-only snapshots from `.planning`; archived apps (e.g. studio, consumer-studio) may live under `apps/*` with ARCHIVED.md or in `legacy/`. **.cursor/plans:** ad-hoc Cursor plans; **.planning** is canonical for phase work (ROADMAP, STATE, TASK-REGISTRY, phases).

**Codebase indexing (when implemented):** Vector store: sqlite-vss. Embeddings: Transformers.js + all-MiniLM-L6-v2; model downloaded during installation (no repo, no first-launch surprise). Excludes `.gitignore`, `vendor/` (by default), `.cursorignore`. See DECISIONS (Codebase indexing, Vector stack).
