# 09 - VideoMode walkthrough (Twick)

This guide shows how **VideoMode** embeds Twick Studio inside the editor
platform, using DockLayout for resizable panels and the same Copilot contract
shape as other modes.

## Goal

- Render Twick Studio in the main panel.
- Provide a right panel for tracks and a bottom panel for the timeline.
- Keep AI tooling consistent with other modes (domain contract + highlights).

## 1. Editor shell + toolbar

VideoMode uses the shared editor shell and slots.

```tsx
// apps/studio/components/modes/VideoMode.tsx
return (
  <EditorShell modeId="video" title="Video" domain="video">
    <ModeHeader>...</ModeHeader>
    <ModeToolbar>...</ModeToolbar>
    <DockLayout ... />
    <ModeStatusBar>...</ModeStatusBar>
  </EditorShell>
);
```

## 2. DockLayout panels

```tsx
<DockLayout
  main={
    <DockPanel panelId="video-main" scrollable={false}>
      <TwickStudio ... />
    </DockPanel>
  }
  right={
    <DockPanel panelId="video-right" title="Tracks">
      <TwickTrackList ... />
    </DockPanel>
  }
  bottom={
    <DockPanel panelId="video-bottom" title="Timeline" scrollable={false}>
      <TwickTimeline ... />
    </DockPanel>
  }
  viewport={{ editorId: 'video-timeline', editorType: 'timeline' }}
  layoutId="video-mode"
/>
```

Expected behavior:
- Panels are resizable and persist layout.
- Video editor remains in the main viewport.

## 3. Domain contract

VideoMode uses `useVideoContract` and `useDomainCopilot` to register
context/actions when the mode is active.

Key file: `apps/studio/lib/domains/video/copilot.ts`.
