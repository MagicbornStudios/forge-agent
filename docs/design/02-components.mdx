# Design: components (atoms -> molecules -> organisms)

This document catalogs the **shared components** we expose for consumers and contributors. It focuses on **real components in the repo** and how they map to our atomic design system.

> **Rule of thumb:** import from `@forge/dev-kit` (or `@forge/ui` / `@forge/shared`) and keep domain code in your own app/mode package.

---

## Atoms (`@forge/ui`)

Shadcn/Radix primitives - always import from the **root** package.

```ts
import {
  Button,
  Input,
  Textarea,
  Badge,
  Card,
  CardHeader,
  CardContent,
  CardTitle,
  CardDescription,
  Item,
  ItemGroup,
  ItemContent,
  ItemTitle,
  ItemDescription,
  Form,
  FormField,
  FormItem,
  FormLabel,
  FormControl,
  FormMessage,
} from '@forge/dev-kit';
```

### Core atoms we use constantly

- **Button / Badge / Card** - structured surfaces and actions.
- **Item / ItemGroup** - list rows and compact summaries (e.g. patch preview items).
- **Form / FormField / FormItem** - React Hook Form helpers (see `CreateNodeModal` for the canonical pattern).

## Molecules (`@forge/shared`)

Composable UI patterns built on atoms.

### Editor chrome

- `EditorApp` (semantic wrapper over AppLayout + tabs + content)
- `EditorShell`
- `ModeHeader` (Left / Center / Right slots)
- `ModeToolbar` (Menubar, Group, Button, ProjectSelect, Settings)
- `DockLayout` (left / main / right / bottom slots)
- `DockPanel` (title, tabs, lock, settings)
- `PanelTabs`
- `PanelSettings`
- `ModeStatusBar`
- `ModeReviewBar`
- `ModeOverlaySurface`
- `EditorButton`, `EditorTooltip`
- `DockSidebar`
- `AppProviders` (optional shared provider stack for tooltips/copilot)

### Gating + AI

- `FeatureGate` (lock/hide/disable)
- `useDomainCopilot`, `DomainCopilotContract`
- `PlanCard`, `PlanActionBar` (generative UI in chat)
- `CodebaseAgentStrategyEditor` (assistant-ui strategy editor)

### Assistant UI + tool UI

- Assistant UI primitives: thread, thread list, markdown renderer, tool fallback.
- Tool UI primitives: plan cards, charts, tables, code blocks, media cards, approval cards.

## Organisms (app + domain)

These live in the **app repo** and are not published.

- `AppShell` - mode tabs + active mode routing.
- `DialogueMode` - React Flow editor + AI workflow panel.
- `VideoMode` - Twick editor surface + AI contract.
- `StrategyMode` - CodebaseAgentStrategyEditor with tool UI.

## Adding components and registries

To add a new shadcn registry or install components from a registry (e.g. `@lucide-animated`, `@billingsdk`), use the correct `components.json`: **packages/ui** for shared components, **apps/studio** or **apps/marketing** for app-only. See [How-to 15 - Shadcn registries and UI components](../how-to/15-shadcn-registries-and-components.mdx) for step-by-step instructions and where components live (shared vs app).

## Consumer guidance

- **Atoms:** import from `@forge/dev-kit` (or `@forge/ui`).
- **Editor shell + AI helpers:** import from `@forge/dev-kit` (or `@forge/shared`).
- **Domain logic:** stays in your app/mode package - not in the shared layer.
