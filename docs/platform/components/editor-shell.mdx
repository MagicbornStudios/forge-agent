# üè† EditorShell

**Root container for all editors with theming and layout skeleton**

The EditorShell is the foundational component for every editor. It provides theming, data attributes for CSS scoping, and a slot-based composition system for organizing editor UI.

---

## Usage

```typescript
import { EditorShell } from '@forge/shared/components/editor';

export function MyEditor() {
  return (
    <EditorShell
      editorId="my-editor"
      title="My Editor"
      subtitle="document-name.txt"
      domain="my-domain"
      theme="dark"
      density="comfortable"
    >
      <EditorShell.Toolbar>
        {/* Toolbar content */}
      </EditorShell.Toolbar>

      <EditorShell.Layout>
        {/* Main layout */}
      </EditorShell.Layout>

      <EditorShell.StatusBar>
        Ready
      </EditorShell.StatusBar>
    </EditorShell>
  );
}
```

## Slot Components

EditorShell uses a slot-based composition system for predictable ordering:

### Available Slots

| Slot | Purpose | Order |
|------|---------|-------|
| `EditorShell.Header` | Optional header section | 1 |
| `EditorShell.Toolbar` | Main toolbar | 2 |
| `EditorShell.Settings` | Settings panel (shares row with toolbar) | 2 |
| `EditorShell.Layout` | Main layout area | 3 |
| `EditorShell.StatusBar` | Bottom status bar | 4 |
| `EditorShell.Overlay` | Modal overlay surface | Float |

### Slot Rendering

When using slots, they render in a fixed order regardless of JSX order:

```typescript
<EditorShell {...props}>
  {/* These render in: Toolbar ‚Üí Layout ‚Üí StatusBar order */}
  <EditorShell.StatusBar>Status</EditorShell.StatusBar>
  <EditorShell.Layout>Layout</EditorShell.Layout>
  <EditorShell.Toolbar>Toolbar</EditorShell.Toolbar>
</EditorShell>
```

## Data Attributes

EditorShell sets the following data attributes for CSS scoping:

```html
<div
  data-editor-id="dialogue"
  data-domain="dialogue"
  data-theme="dark"
  data-density="comfortable"
>
  <!-- Editor content -->
</div>
```

Use these for scoped styling:

```css
[data-editor-id="dialogue"] .my-component {
  /* Dialogue editor specific styles */
}

[data-theme="dark"] .my-component {
  /* Dark theme styles */
}

[data-density="compact"] .toolbar {
  height: 32px;
}
```

## Best Practices

### ‚úÖ Do

- Always provide `editorId` for settings and analytics
- Use slots (`EditorShell.Toolbar`, etc.) for predictable ordering
- Connect theme and density to settings store
- Set `domain` for CSS scoping when needed
- Use `subtitle` to show active document/context

### ‚ùå Don't

- Don't render multiple EditorShells in the same component
- Don't mix slot and non-slot children (use one pattern)
- Don't forget to provide `title` (required for accessibility)
- Don't override flex layout styles (EditorShell uses flex column)
