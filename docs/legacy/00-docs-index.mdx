### 3. Contributor editor requirement

We expect every new contributor to **build their own editor** (tutorial: [05 - Building an editor](how-to/05-building-a-workspace.mdx)) and **submit a PR** to add it under **`packages/shared/contributor_workspaces/`**. See the [README](../README.md) contributor section and [contributor_workspaces README](../packages/shared/contributor_workspaces/README.md).

## I'm a coding agent

1. **Start:** [Agent artifacts index](18-agent-artifacts-index.mdx) and root [AGENTS.md](../AGENTS.md).
2. **Then:** [Coding agent strategy](19-coding-agent-strategy.mdx) - how we use and extend these artifacts, and how to implement your own workflows or benchmarking.

**Always follow this strategy when making changes** (loop, STATUS updates, errors-and-attempts, doc placement). **Quick loop:** STATUS + AGENTS → one slice → update STATUS and affected docs.

Human-facing guides under `docs/` are in MDX where possible for richer in-app reading; living agent artifacts live in `docs/agent-artifacts/core/`. **Doc placement** (where things go: root = entrypoint+policy only; core = living state; no plan docs in core): see [19-coding-agent-strategy.mdx](19-coding-agent-strategy.mdx#doc-placement-where-things-go). **Documentation conventions** (human vs agent, format, number prefixes): see [18-agent-artifacts-index.mdx](18-agent-artifacts-index.mdx#documentation-conventions-human-vs-agent).

**Documentation audiences:** This folder (`docs/`) is for **contributors and in-app docs** (Studio at `/docs`). Consumer-facing docs (Overview, Getting started, Features, Pricing) live in **`apps/platform/content/docs/`** and are shown on the platform site at `/docs`.
