## Phase 2: Model Intelligence (Q2 2026)

**Goal:** Implement smart model routing and cost optimization

### Track: Model Intelligence

- [ ] **Tool complexity classification**
  - Classify all tools as trivial/simple/complex/creative
  - Add metadata to tool definitions
  - Create classification guidelines for new tools

- [ ] **Multi-model routing**
  - Implement `selectModelForOperation()`
  - Create model tier registry (premium/standard/reasoning/free)
  - Add domain-specific model collections
  - Build fallback chains per tier

- [ ] **Prompt caching (Anthropic)**
  - Identify cacheable contexts (world descriptions, project settings)
  - Implement cache_control in system prompts
  - Track cache hit rates
  - Measure cost savings

- [ ] **User model preferences**
  - UI for tier-based model selection
  - Auto-routing toggle
  - Cost preference slider (cost/balanced/quality)
  - Manual override option

- [ ] **Batched operations**
  - `character_createBatch` for multiple characters
  - `forge_createNodeBatch` for bulk node creation
  - Shared context optimization
  - Batch execution UI (progress tracking)

### Track: Infrastructure

- [ ] **Observability**
  - Cost tracking per tool/domain
  - Latency metrics (P50/P95/P99)
  - Model utilization dashboard
  - Cache hit rate monitoring
  - User satisfaction tracking (regeneration rate)

- [ ] **Request optimization**
  - Deduplication for rapid repeated calls
  - Speculative context prefetching
  - Streaming threshold detection (adaptive streaming)

### Track: Capabilities

- [ ] **Storylet model collections**
  - Merchant dialogue model + tone
  - Combat bark model (fast, terse)
  - Lore/mystery model (descriptive)
  - Generic NPC model (budget)

### Success Metrics

- Cost per operation reduced by 70%
- P95 latency < 300ms for trivial operations
- Cache hit rate > 60% for character batch creation
- User cost visibility in settings dashboard

## Phase 3: Multimodal & Advanced (Q3 2026)

**Goal:** Add vision, audio input, and multi-agent capabilities

### Track: Capabilities

- [ ] **Vision integration**
  - Model registry: Add `supportsVision` flag

  - `character_createFromImage` tool
  - `forge_analyzeSceneImage` tool
  - Concept art ‚Üí character generation workflow

- [ ] **Audio input**
  - Speech-to-text integration (Whisper)
  - Voice-first dialogue authoring
  - Full loop: Speech ‚Üí STT ‚Üí LLM ‚Üí TTS
  - Voice command for graph operations

- [ ] **Video generation (future)**
  - Scene visualization from dialogue
  - Character animation preview
  - Model: Sora or equivalent

### Track: Infrastructure

- [ ] **Multi-agent orchestration**
  - Agent swarm pattern (planner/writer/implementer/reviewer)
  - Agent delegation logic
  - Inter-agent communication protocol
  - Shared workspace state

- [ ] **Local edge models**
  - WASM model runtime in browser
  - Trivial operation routing to local models
  - Latency: 10ms vs 500ms
  - Models: `@huggingface/gte-small` for validation

### Track: Model Intelligence

- [ ] **Component-specific models**
  - PlanReviewCard ‚Üí `deepseek-reasoner`
  - CharacterCreatedCard ‚Üí `claude-3.5-sonnet`
  - NodeCreatedCard ‚Üí `gpt-4o-mini`
  - Render-time model selection

- [ ] **Intelligent fallback strategies**
  - Rate limit ‚Üí cheaper model
  - Timeout ‚Üí faster model
  - Context length exceeded ‚Üí larger context model
  - Content policy ‚Üí different provider

### Success Metrics

| Anthropic caching API | Low | Graceful degradation to non-cached |
