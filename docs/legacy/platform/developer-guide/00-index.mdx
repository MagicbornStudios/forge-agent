## ğŸ¨ Design Principles

### 1. Composition Over Configuration

Build editors by composing small, focused components:

```typescript
// âœ… Good - Composable
<EditorShell>
  <EditorToolbar>
    <Button>New</Button>
  </EditorToolbar>
  <DockLayout>
    <DockPanel>Content</DockPanel>
  </DockLayout>
</EditorShell>

// âŒ Bad - Monolithic configuration
<Editor
  config={{
    toolbar: { buttons: [...] },
    panels: [...],
    ...
  }}
/>
```

### 2. Scoped Settings

Settings inherit from app â†’ project â†’ editor â†’ viewport:

```typescript
// App-level default
setSetting('app', 'ui.theme', 'dark');

// Editor-specific override
setSetting('editor', 'ui.theme', 'light', { editorId: 'dialogue' });

// Viewport-specific override
setSetting('viewport', 'ui.theme', 'auto', {
  editorId: 'dialogue',
  viewportId: 'narrative'
});
```

### 3. Selection-Driven UI

The inspector shows properties based on what's selected:

```typescript
const [selection, setSelection] = useState<Selection | null>(null);

// User clicks a node in the graph
setSelection({
  type: 'entity',
  entityType: 'forge.node',
  id: 'node_123'
});

// Inspector automatically shows node properties
<EditorInspector
  selection={selection}
  sections={inspectorSections}  // Filtered by selection.entityType
/>
```

### 4. AI-Augmented Workflows

AI is integrated at the workflow level, not just as a chatbot:

```typescript
// Domain contract defines AI capabilities
const forgeContract = useForgeAssistantContract({
  getInstructions: () => "You help edit dialogue graphs...",
  createTools: () => [
    forge_createNode,
    forge_createEdge,
    forge_createPlan,
  ],
  onHighlight: (entities) => {
    // Highlight nodes created by AI
  },
});
```
