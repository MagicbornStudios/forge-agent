# Forge Platform Developer Documentation

**Complete developer reference** for building editors on the Forge platform. This documentation covers the studio component system, architecture patterns, AI integration, and best practices.

## What is the Forge Platform?

Forge is a **modular, domain-driven editor framework** for building AI-first creative tools. It powers sophisticated editors for:

- **Dialogue** ‚Äî Narrative graph editing (YarnSpinner format)
- **Character** ‚Äî Relationship graph management
- **Video** ‚Äî Timeline-based composition
- **Strategy** ‚Äî Game design planning

## Basic Usage

```tsx
import {
  WorkspaceShell,
  WorkspaceToolbar,
  EditorDockLayout,
  EditorDockPanel,
  WorkspaceStatusBar,
} from '@forge/shared/components/editor';

export function MyEditor() {
  return (
    <WorkspaceShell editorId="my-editor" title="My Editor">
      <WorkspaceToolbar>
        <WorkspaceToolbar.Left>
          <span>My Editor</span>
        </WorkspaceToolbar.Left>
      </WorkspaceToolbar>

      <EditorDockLayout layoutId="my-editor">
        <EditorDockLayout.Main>
          <EditorDockPanel panelId="main" title="Main">
            {/* Your editor content */}
          </EditorDockPanel>
        </EditorDockLayout.Main>
      </EditorDockLayout>

      <WorkspaceStatusBar>Ready</WorkspaceStatusBar>
    </WorkspaceShell>
  );
}
```

## Key Concepts

### 1. Slot-Based Composition

Components use **declarative slots** instead of configuration objects:

```tsx
<WorkspaceShell>
  <WorkspaceShell.Toolbar>...</WorkspaceShell.Toolbar>
  <WorkspaceShell.Layout>...</WorkspaceShell.Layout>
</WorkspaceShell>
```

### 2. Viewport Scoping

Settings and styles scope to **viewports** for multi-viewport editors:

```tsx
const showMiniMap = useSettingsStore((s) =>
  s.getSettingValue('graph.showMiniMap', {
    editorId: 'dialogue',
    viewportId: 'narrative',
  })
);
```

### 3. Data Attributes for Styling

CSS variables cascade via `data-*` attributes:

```tsx
<div
  data-editor-id="dialogue"
  data-domain="dialogue"
  data-viewport-scope="narrative"
>
```

```css
[data-domain="dialogue"] {
  --graph-accent: hsl(210 80% 60%);
}
```

### 4. Domain Contracts for AI

Each editor provides a **domain contract** defining AI tools:

```tsx
const contract = useForgeAssistantContract({
  graph,
  selection,
  applyOperations,
  onAIHighlight,
});

<DialogueAssistantPanel
  contract={contract}
  toolsEnabled={toolsEnabled}
/>
```

## Development Workflow

### 1. Start with the Shell

```tsx
<WorkspaceShell editorId="my-editor" title="My Editor">
  {/* Everything goes here */}
</WorkspaceShell>
```

### 2. Add Toolbar

```tsx
<WorkspaceToolbar>
  <WorkspaceToolbar.Left>
    <span>My Editor</span>
  </WorkspaceToolbar.Left>
  <WorkspaceToolbar.Right>
    <WorkspaceToolbar.Button onClick={onAction}>Action</WorkspaceToolbar.Button>
  </WorkspaceToolbar.Right>
</WorkspaceToolbar>
```

### 3. Add Layout

```tsx
<EditorDockLayout layoutId="my-editor">
  <EditorDockLayout.Left>
    {/* Left sidebar */}
  </EditorDockLayout.Left>
  <EditorDockLayout.Main>
    {/* Main viewport */}
  </EditorDockLayout.Main>
</EditorDockLayout>
```

### 4. Add Status Bar

```tsx
<WorkspaceStatusBar>
  {isDirty ? 'Unsaved changes' : 'Ready'}
</WorkspaceStatusBar>
```

### 5. Add Modals

```tsx
<WorkspaceOverlaySurface
  overlays={overlays}
  activeOverlay={activeOverlay}
  onDismiss={dismissOverlay}
/>
```
