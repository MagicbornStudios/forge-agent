## Settings Codegen Strategy

**UI-First Approach Benefits:**

1. **Declare settings in React** → Auto-generate TypeScript types
2. **Visual development** → See settings as you build them
3. **Type safety** → Generated types for your entire app
4. **Cross-platform** → Export to any system (JSON, YAML, etc.)
5. **Documentation** → Auto-generated from declarations

**Example:**

```typescript
// Declare in UI (developer writes this)
<SettingsSection sectionId="editor">
  <SettingsField
    fieldKey="theme"
    type="select"
    options={['light', 'dark']}
    default="light"
  />
</SettingsSection>

// Generated types (automatic)
interface GeneratedSettings {
  editor: {
    theme: 'light' | 'dark';
  };
}

// Use everywhere (type-safe)
const theme = settings.editor.theme; // 'light' | 'dark'
```
