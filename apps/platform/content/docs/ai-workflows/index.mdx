## What are AI Workflows?

AI Workflows are structured patterns for using AI to accelerate development and content creation on Forge Platform.

### Three Core Workflows

1. **[Cloning](/docs/ai-workflows/cloning)** - Clone and customize existing editors
2. **[Conversations](/docs/ai-workflows/conversations)** - Build through natural language dialogue
3. **[Plans](/docs/ai-workflows/plans)** - AI generates implementation plans you approve

## AI Best Practices

### Give Good Context

**Bad prompt**:
```
"Build an editor"
```

**Good prompt**:
```
"Build a recipe editor with:
- Left panel: category tree (breakfast, lunch, dinner)
- Center: recipe cards with images
- Right: ingredient list and instructions
- AI feature: generate variations (vegan, gluten-free)
Use @forge/dev-kit components"
```

### Be Specific About Requirements

**Bad**:
```
"Add a form"
```

**Good**:
```
"Add a form with:
- Name (required, max 50 chars)
- Email (validated email format)
- Role (dropdown: viewer, editor, admin)
- Submit button that calls /api/users"
```

### Iterate in Small Steps

**Bad approach**:
```
"Build entire application with all features"
```

**Good approach**:
```
1. "Set up EditorShell with basic layout"
2. "Add left sidebar with category tree"
3. "Add main panel with item cards"
4. "Add inspector panel for details"
5. "Connect to API and add CRUD operations"
```

### Review and Refine

**Workflow**:
1. AI generates code
2. You review for correctness
3. Test functionality
4. Request refinements
5. Iterate until satisfied

## AI Integration Points

### During Development

**Component Selection**:
```
You: "I need a modal dialog"
AI: "Use <Dialog> from @forge/ui with DialogContent and DialogTitle"
```

**Code Generation**:
```tsx
You: "Generate a user profile form"
AI: <generates>
<Form>
  <Input name="name" label="Full Name" required />
  <Input name="email" type="email" label="Email" required />
  <Textarea name="bio" label="Bio" rows={4} />
  <Button type="submit">Save Profile</Button>
</Form>
```

**Debugging**:
```
You: "This component isn't updating"
AI: "Check if you're using useState for the value.
     Also make sure onChange is calling the setter."
```

### During Content Creation

**Dialogue Generation**:
```
You: "Generate merchant dialogue, friendly but greedy"
AI: <generates Yarn Spinner dialogue with personality>
```

**Character Creation**:
```
You: "Create a rival character for my protagonist"
AI: <generates character profile with conflicts and motivations>
```

**Documentation**:
```
You: "Document this component's props"
AI: <generates JSDoc comments with examples>
```

## Security and Privacy

### What AI Can Access

**‚úÖ Allowed**:
- Your current project structure
- Component library documentation
- Your explicitly provided context
- Public Forge documentation

**‚ùå Not Allowed**:
- Other users' projects
- Private API keys
- Sensitive user data
- Production databases

### Data Handling

**During generation**:
- AI processes your prompts and context
- Generated code is yours to keep
- No training on your private data

**Privacy controls**:
- Opt out of AI features anytime
- Delete conversation history
- Control what context AI can access
