# Forge Developer Program

**Build once, sell everywhere. Earn 70% revenue share on your custom editors.**

The Forge Developer Program is like the Unity Asset Store for narrative tools. Build custom editors using our dev-kit, submit them for review, and earn revenue when teams use your creations.

## What is the Developer Program?

The Developer Program enables developers to create and monetize custom editors within the Forge ecosystem.

### How It Works

1. **Build** custom editors using `@forge/dev-kit`
2. **Test** with real projects to ensure quality
3. **Submit** for technical and quality review
4. **Publish** to the Forge Studio marketplace
5. **Earn** 70% of revenue from sales and subscriptions

### Who Should Join?

- **Tool developers** building workflow solutions
- **Agencies** creating client-specific editors
- **Game studios** with reusable narrative tools
- **Indie developers** monetizing their expertise
- **Technical writers** building specialized content tools

## Submission Process

### Phase 1: Development

**1. Set up your environment**

```bash
# Install Forge dev-kit
npm install @forge/dev-kit

# Create your editor
mkdir my-custom-editor
cd my-custom-editor
npm init -y
```

**2. Build your editor**

Use Forge components and patterns:

```typescript
import {
  EditorShell,
  EditorDockLayout,
  EditorDockPanel,
  EditorToolbar,
} from '@forge/dev-kit';

export function MyCustomEditor() {
  return (
    <EditorShell
      editorId="my-custom-editor"
      title="My Custom Editor"
    >
      <EditorToolbar>
        {/* Your custom toolbar */}
      </EditorToolbar>

      <EditorDockLayout layoutId="custom-layout">
        <EditorDockLayout.Main>
          <EditorDockPanel id="main" title="Main View">
            {/* Your editor UI */}
          </EditorDockPanel>
        </EditorDockLayout.Main>
      </EditorDockLayout>
    </EditorShell>
  );
}
```

**3. Test thoroughly**

- Create sample projects
- Test all features
- Verify data persistence
- Check performance
- Test AI integrations (if applicable)

## Requirements & Guidelines

### Technical Requirements

**Must use `@forge/dev-kit`**

All editors must be built on our component library:

```typescript
// ‚úÖ Good - uses Forge components
import { EditorShell, DockLayout } from '@forge/dev-kit';

// ‚ùå Bad - completely custom implementation
import { CustomShell } from './my-custom-lib';
```

**TypeScript strongly recommended**

Type-safe editors are easier to maintain:

```typescript
import type { EditorConfig, PanelConfig } from '@forge/dev-kit/types';

interface QuestEditorState {
  quests: Quest[];
  selectedQuest: string | null;
}
```

**Performance benchmarks**

- Initial load: < 2 seconds
- Panel rendering: < 100ms
- Data save: < 500ms
- Memory usage: < 200MB for typical projects

**Browser compatibility**

Support latest versions of:
- Chrome/Edge (Chromium)
- Firefox
- Safari

**Accessibility**

- Keyboard navigation
- Screen reader support
- WCAG AA compliance
- High contrast mode support

### Design Guidelines

**Follow Forge design language**

Use provided components:
- Buttons, inputs, dropdowns from `@forge/dev-kit/ui`
- Theme variables for colors
- Standard spacing and typography
- Consistent iconography

**Responsive layouts**

- Support window resizing
- Panels collapse gracefully
- Mobile-friendly (if applicable)

**Dark/light theme support**

Inherit from EditorShell:
- No hardcoded colors
- Use CSS variables
- Test in both modes

### Documentation Requirements

**User documentation**

Must include:
- Getting started guide
- Feature overview with screenshots
- Common workflows and examples
- Troubleshooting section
- FAQ

**Developer documentation**

If extensible:
- API reference
- Plugin/extension guide
- Code examples
- TypeScript types

**Video tutorials**

Highly recommended:
- 2-5 minute overview
- Feature walkthrough
- Real-world example

## Frequently Asked Questions

### Do I need to be a company to join?

No. Individual developers, freelancers, and companies can all join. You'll need to provide tax information for revenue payouts.

### Can I sell my editor elsewhere?

Yes, but it must use `@forge/dev-kit` which requires Forge Platform. You can't strip out Forge dependencies and sell standalone.

### What if my editor stops working due to platform updates?

We provide advance notice (minimum 90 days) for breaking changes and detailed migration guides. We work with developers to ensure smooth transitions.

### Can I offer free trials?

Yes. You control pricing, including free tiers, trial periods, and freemium models.

### How do refunds work?

Customers can request refunds within 14 days. We handle refund requests and absorb the cost. You keep your 70% of non-refunded sales.

### Can I update my editor after publication?

Yes. Push updates anytime through the Developer Portal. Users get automatic updates. We recommend a changelog and version numbering.

### What happens if I abandon my editor?

We'll mark it as "no longer maintained" after 6 months of no updates. It remains available for purchase, but with a clear notice. You continue earning revenue.

### Can I build proprietary editors for clients?

Yes. You can build private editors not listed in the marketplace. Client pays you directly, they get exclusive access. No revenue share with Forge.

### Do you take a cut of client work?

No. If you're building custom editors for clients under contract, you keep 100%. Revenue share only applies to marketplace sales.
