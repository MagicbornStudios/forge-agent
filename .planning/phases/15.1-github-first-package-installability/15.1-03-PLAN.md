# Plan 15.1-03: App and outlier install surfaces

## Goal

Close the gap between "package manifests are sane" and "everything is installable from GitHub" by explicitly deciding the install surface for apps and the packages that still need repo splits or submodules.

## Scope

- Define which app surfaces are consumed by repo clone, dedicated repo, or submodule.
- Identify any package that still cannot be consumed directly after manifest hardening.
- Feed those outcomes into Phase 16 and later repo-split work without reintroducing npm as a dependency.
