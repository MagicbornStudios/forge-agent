# Plan 15.1-01: Package manifest normalization

## Goal

Remove package metadata assumptions that only work inside the local pnpm workspace. This plan hardens package manifests so GitHub installation is practical and predictable.

## Scope

- Replace `workspace:*` dependency specs with GitHub-safe `file:` paths for in-repo dependencies.
- Review `private` flags on reusable packages and remove them where they are blocking GitHub install use.
- Add `prepare` hooks for packages whose entrypoints depend on generated `dist` artifacts.
- Fix mismatches between `exports` and `files`, especially where `src` subpaths are exported.
- Fix missing subpath exports that currently rely on workspace-only resolution behavior.

## Verification

- No `workspace:*` references remain in `apps/*/package.json` or `packages/*/package.json`.
- `@forge/types` supports existing subpath imports such as `@forge/types/graph`.
- Packages that rely on `dist` outputs define a GitHub-safe install build hook.
- Packages exporting `src` files include those files in their package allowlist.

## Done criteria

- Core manifests are GitHub-safe enough for smoke verification.
- Existing local workspace development still installs successfully.
- Remaining install blockers are narrowed to app/repo topology decisions, not obvious manifest bugs.
