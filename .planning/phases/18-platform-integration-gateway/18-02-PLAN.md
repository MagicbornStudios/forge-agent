# Plan 18-02: Extension install via platform

## Goal

When the user has no GitHub token, Repo Studio can still install extensions by calling the platform. Platform fetches extension content from the RepoStudio-Extensions repo (using a platform-held GitHub token) and returns content or a signed URL to Repo Studio.

## Scope

- Platform: API that accepts extension install requests (e.g. extension id/ref), uses platform-held token to fetch from RepoStudio-Extensions (e.g. GitHub API or raw content), returns package content or signed URL.
- Repo Studio: when `capabilities.extensionInstallProxy` is true and user has no GitHub auth, use platform extension-install endpoint instead of direct GitHub fetch.
- Human sets GitHub token for RepoStudio-Extensions fetch in Vercel (HUMAN-TASKS HT-06).

## Out of scope

- Private or user-specific extension repos (only public/canonical RepoStudio-Extensions in this plan).
- Changing extension catalog or manifest format (existing .repo-studio/extensions and registry APIs).
