# Plan 20-03: On-load planning prompt and optional notifications

## Goal

When the user opens the **Planning workspace**, optionally **open the assistant** and send a **planning-focused prompt** (e.g. "What should we work on next?" or "Review current state and suggest next steps") to Codex so the assistant can discuss what to do without the user typing first. Optionally add **tagging or notifications** for human-blocked items (e.g. rail badge when HUMAN-TASKS has open items).

## Scope

- **On-load behavior:** On Planning workspace mount (or first focus), optionally: open assistant panel if closed, switch to Codex if not already, send a system or user message that triggers a planning "what to do" response. Make this **configurable** (e.g. setting or feature flag) so we donâ€™t force the behavior on every load.
- **Tagging/notifications:** Optional: badge or indicator when HUMAN-TASKS has open (incomplete) items; could be in rail, or in the Human TODOs panel header. Document in DECISIONS if deferred.

## Out of scope

- Changing shared AssistantPanel contract (only Repo Studio Planning workspace behavior).
- Mandatory auto-send on every load without a way to disable.
